# Designer Q&A

## Can I submit only a Figma link?

No. Include scope, frames, breakpoints, states, assets, interactions, assumptions, and open questions.

## Must I design desktop, tablet, and mobile?

Usually provide desktop and mobile. Provide tablet when it differs materially. Otherwise state the tablet inheritance rule.

## Must every hover and focus state be drawn?

No. Existing design-system or code component defaults can be referenced. Custom controls and critical CTAs need explicit states.

## Must loading, empty, and error states be designed?

If users can encounter them, they must be drawn, referenced from an existing pattern, or explicitly delegated as developer-allowed assumptions.

## What if copy is not final?

Mark final versus placeholder copy, give max length or edge examples, and note localization risk when relevant.

## What if final images are unavailable?

Provide aspect ratio, focal point, crop rule, responsible owner, and whether placeholders are allowed.

## Can I introduce new colors or type values?

Yes, but name whether they are new tokens, local one-off values, or design-system changes.

## How much annotation is enough?

Annotate only development-critical decisions: responsive rules, state behavior, asset handling, and interaction outcomes.

## Can the design change after review?

Yes. Small copy or asset changes may need incremental review. Structure, interaction, state, or responsive changes need handoff review again.

## Can Conditional designs enter build?

Yes, only when blockers are resolved and all assumptions are explicit, safe, and carried into build notes.
