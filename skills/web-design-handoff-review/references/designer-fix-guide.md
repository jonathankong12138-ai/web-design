# Designer Fix Guide

Use these instructions when writing `Designer Instruction` items.

## Scope

Ask the designer to add a handoff page, list ready frames, move old versions to archive, and state in-scope and out-of-scope work.

## Breakpoints

Ask for a mobile frame, or for explicit rules covering content order, column changes, media crop, navigation collapse, sticky behavior, and tablet inheritance.

## States

Ask for missing default, hover, focus, active, disabled, selected, loading, success, error, empty, open, close, overflow, and permission states. If existing component states are acceptable, ask the designer to name that component or pattern.

## Content

Ask for final copy or placeholder labels, representative sample data, max length, empty values, field source, CMS/API ownership, and localization notes.

## Tokens

Ask the designer to use design-system variables, name new tokens, explain one-off values, and align inconsistent spacing or type scales.

## Assets

Ask for export-marked image/icon/logo layers, vector icons, image aspect ratio, crop focal point, mobile treatment, font source, license, and fallback.

## Interactions

Ask for click targets, destination, modal/drawer behavior, validation, submit success/failure, scroll behavior, carousel/tabs state, and motion trigger/duration/end state.

## Prototype Conflict

Ask for a decision note explaining why the design differs from prototype, who approved it, and whether build should follow prototype or design.
