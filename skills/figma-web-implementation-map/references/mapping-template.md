# Implementation Mapping Template

Use this only long enough to guide code.

```txt
Route Mapping:
- Design frame:
- Existing route / page / template:
- New route needed:

Component Mapping:
- Design module -> existing component:
- Design module -> new component:
- Props / slots / data:

Token Mapping:
- Raw design value -> existing token:
- Raw design value -> new local value:
- Values intentionally approximated:

Asset Mapping:
- Asset:
- Source:
- Destination:
- Format / compression:
- Fallback:

Responsive Strategy:
- Desktop:
- Tablet:
- Mobile:
- Breakpoint transitions:
- Content order changes:

Implementation Slices:
- Slice 1:
- Slice 2:
- Slice 3:

Assumptions:
- Missing state:
- Missing data:
- Missing asset:
- Product decision:
```

Prefer existing tokens and components. If a design value conflicts with the local system, record the raw value and the mapped local value.
