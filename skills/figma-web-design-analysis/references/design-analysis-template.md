# Design Analysis Template

Use this template when the design analysis must guide later implementation.

```txt
Design Source:
- Type:
- Link / file / screenshot:
- Access limits:

Frames / Breakpoints:
- Desktop:
- Tablet:
- Mobile:
- Other states or overlays:

States:
- Default:
- Hover / focus / active:
- Loading / empty / error:
- Selected / expanded / disabled:

Page Structure:
- Header / nav:
- Main sections:
- Repeated modules:
- Forms / controls:
- Footer:

Content:
- Headings:
- Body copy:
- Buttons and links:
- Labels and fields:
- Example data:

Design Tokens:
- Colors:
- Typography:
- Spacing:
- Radius:
- Shadows / borders:
- Layering:

Assets:
- Images:
- Icons / logos:
- Video:
- Fonts:
- Export or fallback notes:

Interactions:
- Click / navigation:
- Inputs:
- Overlay behavior:
- Scroll / carousel / tabs:
- Motion:

Unknowns:
- Missing source:
- Missing state:
- Missing token:
- Missing asset:
- Assumption:
```

Keep raw values when they matter, but do not over-extract every pixel. Prioritize values that affect implementation decisions or visual QA.
