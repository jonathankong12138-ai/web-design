# Web Design

`web-design` is a four-stage Codex plugin for web app work. Each stage has exactly one allowed artifact.

## Workflow

| Stage | Skill | Only artifact |
|---|---|---|
| Requirements | `prototype` | Interactive prototype |
| Development | `build` | Code matching the prototype and current checks |
| Testing | `check` | Developer adjustment checklist |
| Retrospective | `retro` | Codex-ready skill upgrade guide |

## Principle

All product, design, engineering, testing, and review methods are internal tools. They must improve the stage artifact, not create extra deliverables.

## Use

Ask Codex to use one of:

- `prototype` to turn a brief, PRD, feature request, or vague idea into a runnable prototype.
- `build` to implement code from the prototype and latest adjustment checklist.
- `check` to inspect the app and return a concrete adjustment checklist for development.
- `retro` to convert task lessons into paste-ready skill upgrade guidance.

## Local Install

Clone or copy this folder into a plugin location Codex can load, then install it as a local plugin or point Codex at the plugin directory.
